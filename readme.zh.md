# 一个可交互的狭义相对论模拟

---

## 基础用法

1. 前往 [莉莉丝页面] 里下载 `special-relativity-windows-x86_64-pack.tar.xz`（linux 用户为 `special-relativity-linux-x86_64-pack.tar.xz`），

2. 解压到任意文件夹并运行 `special-relativity.exe`（linux 用户为 `special-relativity`）。

    linux 用户需要在系统里安装 glfw3。

---

## 系统要求

- glfw 3.4 和 vulkan 1.3 兼容性（人话说就是有2022年之后显卡驱动更新的电脑）。

---

## 操作

- 移动镜头：移动鼠标，

- 移动（加速）：`WASD` 前左后右，`ctrl` 下，`空格` 上，

- 调整推力：鼠标滚轮，

- 退出：`alt`+`F4`。

## 画面真实性

### 天球

- 天球假设静态并且距“玩家”无限远，也就是说无论移动多快还是时间经过多久，天球本身是不会变化的。

- 天球是由自制的（效果稀烂的）算法生成的，移动时星体会非常闪烁，而且接近光速运动时仍然会有大片区域保持黑暗。

    为了计算光的多普勒效应，天球需要提供“全频”光谱辐射率，或者，在这里实现里是使用亮度和色温计算黑体辐射。

    使用现实数据作为天球渲染是可能的（见 [我的文章](https://zhuanlan.zhihu.com/p/600600997)），但这会使莉莉丝变得巨大，程序变得复杂，还有版权什么的。

### 后处理

没有，除了一个 tone-maping，而这会导致一个很明显的“错误”：没有炫光。
在接近光速前进时，前进方向的入射光会变得非常闪光弹，现实来说这会导致非常强烈的炫光并且漂白前方所有景象，
而这里没有炫光就会导致接近光速移动时看起来很平淡。

没有后处理同样会导致没有“自动曝光”，而且我也没加手动曝光。

## 进阶用法

程序本身（`special-relativity.exe` 或 `special-relativity`）是不足以运行的，还需要提供外部的着色器文件，由此用户可以提供自定义着色器。在空白文件夹里以无启动参数运行程序会生成这里使用的着色器。

### 参数

- `-s="path/to/shaders"` 或 `--shader="path/to/shaders"`：指定着色器文件夹，默认为当前运行路径，文件夹里必须存在 spirv 着色器文件 `init_ray.spv` 和 `render_ray.spv`
（或 slang 着色器文件，见下）。

- `--slangc="path/to/slangc"`: 指定 slang 编译器路径，如果着色器文件夹内不存在 spirv 着色器文件但存在 slang 着色器文件 `init_ray.slang` 和 `render_ray.slang`，
程序会自动调用 slangc 编译 slang 到 spirv。
